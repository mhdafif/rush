01 Context
Currently, history and personal best are only visible on the results screen — the screen that appears after finishing a test. There is no way to check them from the main (idle) screen without completing a test first.

Three options are on the table. Each offers a different trade-off between discoverability, complexity, and disruption to the existing minimalist layout.

idle
→ start typing →
running
→ finish →
results
← history only visible here
02 Option A — Stats Modal overlay
A
Stats Modal
A small icon button in the header opens a centered modal overlay with history + best score.

Layout Wireframe

┌────────────────────────────────────────────┐ │ CYBERTYPE EN │ ID 15s 30s 60s [📊] │ ← icon in header ├────────────────────────────────────────────┤ │ │ │ ░░░░░░░░░░░░░░░░░░░░░░░ │ │ ░░░ typing arena ░░░░ │ │ ░░░░░░░░░░░░░░░░░░░░░░░ │ │ │ └────────────────────────────────────────────┘ ┌── when icon clicked ───────────┐ │ STATS & HISTORY × │ │ ───────────────────────────── │ │ Personal Best 142 WPM 97% │ │ │ │ Recent Tests │ │ ① 30s 142 wpm 97% EN │ │ ② 60s 138 wpm 95% EN │ │ ③ 15s 130 wpm 99% ID │ │ ④ 30s 125 wpm 93% EN │ │ ⑤ 60s 120 wpm 90% ID │ └────────────────────────────────┘

Pros

  • Main layout completely unchanged
  • Accessible any time (idle, paused, results)
  • Header icon is a familiar UX pattern
  • Minimal file changes — one new component
  • Does not add a new app "view" or route

Cons

  • Modal feels slightly heavy for a minimalist app
  • Icon in header may clutter the clean bar
  • Backdrop dims the typing arena
  • Needs Esc handling (conflict with reshuffle?)

Trigger & Dismiss

Action Behaviour
Click [📊] icon in header Opens modal
Click backdrop Closes modal
Esc Closes modal (idle only — during test, Esc still reshuffles)
Click × button Closes modal
03 Option B — Sidebar Drawer drawer
B
Sidebar Drawer
A panel slides in from the right side of the screen, pushing or overlaying the main content.

Layout Wireframe

┌──────────────────────────────┬──────────────┐ │ CYBERTYPE EN│ID 15s 30s [≡] │ STATS × │ ├──────────────────────────────┤ ──────────── │ │ │ Best │ │ ░░░░░░░░░░░░░░░░░ │ 142 WPM 97% │ │ ░░ typing arena ░░ │ │ │ ░░░░░░░░░░░░░░░░░ │ History │ │ │ ① 142w 97% │ │ │ ② 138w 95% │ │ │ ③ 130w 99% │ │ │ ④ 125w 93% │ │ │ ⑤ 120w 90% │ └──────────────────────────────┴──────────────┘

Pros

  • Content stays visible alongside the test
  • No backdrop — typing arena stays bright
  • Natural scroll for long history lists
  • Feels modern and non-intrusive

Cons

  • Narrows the typing arena when open
  • More CSS complexity (slide animation, overflow)
  • Can feel cramped on smaller screens
  • Two more components needed (Drawer + DrawerContent)

Trigger & Dismiss

Action Behaviour

Test starts (keypress) Drawer auto-closes to give full width
04 Option C — Separate View view
C
Separate View
A third app "screen" (alongside typing and results) dedicated entirely to history and stats.

App View Flow

→ finish →
results
→ retry →
typing
typing
→ H or header link →
history view
→ back →

Layout Wireframe

┌────────────────────────────────────────────┐ │ CYBERTYPE EN │ ID 15s 30s 60s │ │ [History] │ ← nav link ├────────────────────────────────────────────┤ │ │ │ Personal Best ────────────────────────── │ │ 142 WPM · 97% Accuracy · 60s · EN │ │ │ │ Recent Tests ─────────────────────────── │ │ # Mode WPM Acc Lang Date │ │ 1 60s 142 97% EN Apr 27 │ │ 2 30s 138 95% EN Apr 27 │ │ 3 15s 130 99% ID Apr 26 │ │ 4 30s 125 93% EN Apr 26 │ │ 5 60s 120 90% ID Apr 25 │ │ │ │ [← Back to typing] │ └────────────────────────────────────────────┘

Pros

  • Cleanest history display — full width, full context
  • Easiest to extend (charts, filters, export later)
  • No overlay/backdrop complexity
  • No layout shift on the typing screen
  • Could add a WPM trend chart in future

Cons

  • Navigates away from the test — higher friction
  • More code: new view component + view state
  • Requires AppView type to gain a third value
  • Header needs a new persistent nav link

07 Recommendation suggested
Recommended: Option A (Stats Modal) — lowest effort, zero layout disruption, and fits the minimalist cyberpunk aesthetic. The Esc conflict is trivially handled by only intercepting it when the modal is open.
Best for simplicity
A — Modal
Minimal code, no layout change, quick to ship.
Best for aesthetics
B — Drawer
Sleek slide-in feels most cyberpunk, good side panel.
Best for future growth
C — Separate View
Full screen, easy to add charts/filters later.

Pick whichever option fits your preference — all three are well-scoped and implementable in a single session. Let me know which one you want and I'll build it.
