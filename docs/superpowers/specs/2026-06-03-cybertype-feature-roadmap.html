CYBERTYPE Feature Roadmap
2026-06-03 React + Vite Zustand Low-risk first
01Overview
Goal: extend CYBERTYPE with useful, incremental features while preserving the current clean SPA architecture.
First priority
Improve local history and personal best handling. These are visible, useful, and low-risk.
Second priority
Add practice-oriented improvements that use data already collected by the typing engine.
Third priority
Add personalization and retention features once the core stats model is stronger.

03Low Effort Featuresstart here
FeatureValueImplementation Notes
Clear history optionLets users reset local stats safely.Add clearHistory action in settingsStore. Add a button in StatsDrawer with a confirmation step. Decide whether it clears only history or both history and best.
Per-mode personal bestsPrevents a 15s time score from competing with quotes or 100-word tests.Replace single best with a keyed best map by mode:duration:lang or similar. Keep migration minimal because the app is local-only.

Export historyUseful for backup and manual analysis.Add JSON first. CSV can come later. This should be a simple browser download from stored history.
04Medium Effort Features
Practice weak keys mode
Use the existing error heatmap data to generate drills focused on keys the user misses most often.
More themes / light mode
Add a persisted theme setting and CSS variable sets. Start with current cyber orange, light mode, and one alternate accent.
Font size / density settings
Let users tune word display size and spacing without changing typing logic.
More languages
Add additional word lists and quotes. Spanish, French, and German are good candidates.
Zen mode
Hide live stats while typing and only show results after completion.
Full history table
Add filtering by mode, language, date, and personal-best status.
05Large Features
  • Daily streaks: Track practice days using local dates and history timestamps.
  • Achievements / XP: Reward milestones like first 100 WPM, 95% accuracy, or 7-day streak.
  • Keyboard layout options: Support QWERTY, Colemak, Dvorak, and maybe custom layouts for heatmap rendering.
  • Backspace across word boundary: Improve typing ergonomics but requires careful changes to the word state machine.
  • Accounts / cloud sync: Only consider this after the local stats model stabilizes.

08Verification
  • Run tests: Use the project test command from package.json after each feature.
  • Manual history test: Complete multiple tests, reload the page, confirm history persists, clear it, reload again.
  • Manual bests test: Complete different modes/durations and confirm bests do not overwrite unrelated categories.
  • Manual HUD test: Check time, words, and quotes modes for correct progress display.
  • Regression pass: Verify restart, retry same, new test, pause, sudden death, and mute still work.
