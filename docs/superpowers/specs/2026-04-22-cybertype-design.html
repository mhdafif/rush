06 Word Lock Behavior
When Space is pressed: (1) evaluate typed input vs expected word, (2) set state to locked-correct or locked-error, (3) advance cursor. The locked word cannot be edited — backspace cannot reach it.

Backspace Rules

SituationResult
Cursor inside word (typed chars exist)✓ Delete last typed char in current word
Cursor at word start (nothing typed yet)✗ Blocked — no-op, cannot reach previous word
Previous word is locked✗ Blocked — locked words are immutable

handleKeyDown Logic

// useTypingEngine — handleKeyDown
if (isPaused || phase !== 'running') return

if (key === ' ') {
  lockCurrentWord()   // → 'locked-correct' or 'locked-error'
  if (isLastWord && mode === 'words') {
    endTest()           // Space on final word ends the test immediately
  } else {
    advanceToNextWord()
  }
}

if (key === 'Backspace') {
  if (currentInput.length === 0) return  // blocked at word boundary
  deleteLastChar()
}
07 Pause / Resume Behavior ★ New
Pause only activates during an active test (phase = running). Has no effect during idle or results view.

State Machine

idle
→
first keypress
→
running
→
mouse moves or window blur
→
paused
→
click arena or any key
→
running

When Paused

  • Timer frozen — no tick
  • Keydown events ignored by engine
  • Arena content: filter: blur(5px)
  • PauseOverlay renders over arena
  • Live stats strip dims to low opacity

To Resume

  • Click anywhere on the typing arena
  • Press any typeable key
  • Both re-focus the hidden input
  • Timer resumes from frozen value
  • Blur transitions out via CSS transition

useMousePause Hook

useEffect(() => {
  const handleMouseMove = () => {
    if (phase === 'running') setIsPaused(true)
  }
  const handleBlur = () => {
    if (phase === 'running') setIsPaused(true)  // tab/window loses focus
  }
  window.addEventListener('mousemove', handleMouseMove)
  window.addEventListener('blur', handleBlur)
  return () => {
    window.removeEventListener('mousemove', handleMouseMove)
    window.removeEventListener('blur', handleBlur)
  }
}, [phase])

// isPaused is passed to both useTimer and useTypingEngine — both halt when true
08 Timer Hook (useTimer.js)
ModeBehavior
TimeCounts down from duration. Fires onEnd when it reaches 0. Pauses when isPaused = true.
WordsCounts up from 0. Stops when engine signals test complete. Pauses when isPaused = true.
Uses setInterval cleared on pause and on unmount. Every second tick the engine records a wpmHistory snapshot.
09 WPM Calculation 
// elapsedSeconds mapping:
//   time mode  → duration - remainingSeconds  (e.g. 60 - 38 = 22s elapsed)
//   words mode → count-up timer value

wpm      = (correctChars / 5) / (elapsedSeconds / 60)
rawWpm   = (totalTypedChars / 5) / (elapsedSeconds / 60)
accuracy = (correctChars / totalTypedChars) * 100
Retry Same: testResult, wpmHistory, and all engine state are fully reset before the new test begins. The same word list array is reused.

13 Focus Management
  • A hidden <input> inside TypingArena captures all keyboard events
  • Clicking the arena triggers .focus() on the hidden input
  • On resume (click or keypress while paused), hidden input is re-focused
  • The word display area is user-select: none and pointer-events: none
