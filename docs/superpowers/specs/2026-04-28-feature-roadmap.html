02 Game Modes
BUG FIX
Tab Key — Accidental Restart Guard
✓ done highS
Tab previously triggered a test restart unconditionally — including during idle and active typing, causing accidental restarts. Now Tab only restarts when the test is paused. The restart shortcut hint icon is also hidden during active typing.

Custom Time Duration
✓ done highS
Let users type a custom number for the timer instead of only 15 / 30 / 60s. A small inline input in the mode selector lets them enter any duration (e.g. 45s, 90s, 120s) and confirm with Enter. Value clamps to 10–300s.

F-02
Punctuation & Numbers Mode
✓ done highM
Toggle buttons to include punctuation and/or numbers in the generated word list. When punct is on, ~25% of words get a comma, period, or apostrophe appended. When num is on, ~15% of words are replaced with random 1–4 digit numbers. Both can be active simultaneously.

F-03
Sudden Death Mode
✓ done mediumS
The test ends immediately on any single error. A toggle in the mode selector activates it. When active, the typing arena shows a subtle red-tinted border. On failure, results show "TERMINATED" with WPM at time of death and word reached.

Quotes Mode
mediumL
A new mode alongside time / words that loads a random quote from a bundled dataset. The test ends when the last character is typed — no timer. Quotes are sourced from a static quotes.ts file (20–30 entries per language). Short / medium / long length filters can be added later.

F-09
Sound Effects
✓ done mediumM
Subtle audio feedback: soft click on correct keypresses, a distinct blip on errors, chime on test completion. Uses the Web Audio API — no libraries, no audio files. Sounds generated procedurally as short sine/square wave tones. A mute toggle (persisted) appears in the stats drawer.

F-10
Keyboard Heatmap
lowL
After a test completes, the results screen shows a QWERTY keyboard layout where each key tile is heat-colored based on the number of times that key was typed incorrectly during the test.

Visual layout: Three staggered rows of key tiles (Row 1: Q–P, Row 2: A–L, Row 3: Z–M), rendered as flex rows with slight left-offset per row to mimic a real keyboard. Each key is a small square tile (~32×32px), rounded corners, font-mono uppercase letter centered inside.

Color tiers by error count:
 • 0 errors → bg-zinc-800 text-zinc-500 (neutral/dark)
 • 1–2 errors → bg-orange-950/60 text-orange-400 (mild heat)
 • 3–5 errors → bg-orange-900/70 text-orange-300 (warm)
 • 6+ errors → bg-orange-600 text-white (hot — full glow)

The heatmap is shown only when there is at least one error. It appears on the results screen below the WPM chart in a card labeled "key errors". Keys with errors also show a small error-count badge in the corner.
useTypingEngine.ts — track keyErrors map ResultsScreen.tsx — render heatmap card new file: KeyboardHeatmap.tsx
