01 Goal & Principles

Goal: Refactor CYBERTYPE so every screen file is a thin view, all logic lives in a paired use*.tsx hook, shared state lives in Zustand stores (no prop drilling), and history/settings persist via localStorage through Zustand middleware.

Convention from ex/login.tsx + ex/useLogin.tsx: each view imports one hook; the hook is sectioned by /*=== Props ===*/, /*=== Store ===*/, /*=== UseState ===*/, /*=== Handler ===*/, /*=== UseEffect ===*/, /*=== Return ===*/.
  • No prop drilling — all cross-component state consumed directly from stores.
  • Micro-components — large files broken into focused sub-components, one responsibility each.
  • Zustand stores — two stores: useTypingStore (test/engine state) and useSettingsStore (config + history + best, persisted to localStorage).
  • Existing hooks preserved — useTypingEngine, useTimer, useSound, etc. keep their internals; they feed the store.
  • Tests still pass — pnpm validate green after every task.
  • Commits per task — one focused commit per task.
02 Architecture Overview

Before

App.tsx destructures ~20 values from useApp and passes them as props 2–3 levels deep. Components like Header receive 13 props.

After

App.tsx renders layout only. Each sub-component reads exactly what it needs from a store. useApp is split into feature hooks.

Data flow (after)

// Zustand stores — all components subscribe directly

useSettingsStore  → visualMode, duration, lang, caretStyle, suddenDeath, muted
                  → history[], best  (persisted via localStorage middleware)

useTypingStore    → phase, words, wordStates, currentInput, …
                  → testResult, view ("typing" | "results")
                  → isPaused, wpm, accuracy, timeDisplay

04 Store Shapes

useSettingsStore — persisted to localStorage

// src/store/settings/ISettingsStore.ts
export interface ISettingsState {
  visualMode: VisualMode;        // "time" | "words" | "punct" | "nums" | "quotes"
  duration:   number;
  lang:       Language;           // "en" | "id"
  caretStyle: CaretStyle;         // "line" | "underline"
  suddenDeath:boolean;
  muted:      boolean;
  history:    HistoryEntry[];     // last 5 results
  best:       BestScore | null;
}
export interface ISettingsStore extends ISettingsState {
  setVisualMode: (vm: VisualMode) => void;
  setDuration:   (d: number)     => void;
  setLang:       (l: Language)    => void;
  setCaretStyle: (s: CaretStyle)  => void;
  toggleSuddenDeath: ()            => void;
  toggleMute:    ()                => void;
  saveResult:    (r: TestResult, lang: Language) => void;
}

useTypingStore — in-memory only

// src/store/typing/ITypingStore.ts
export interface ITypingState {
  view:        "typing" | "results";
  words:       string[];
  quoteAuthor: string | undefined;
  isPaused:    boolean;
  testResult:  TestResult | null;
  wpm:         number;
  accuracy:    number;
  timeDisplay: string;
  resetKey:    number;
  isDrawerOpen:boolean;
}
export interface ITypingStore extends ITypingState {
  setState: <K extends keyof ITypingState>(key: K, value: ITypingState[K]) => void;
  reset:    () => void;
}
Persist strategy: settingsStore uses Zustand persist middleware with localStorage. Keys: cybertype_settings. History and best are stored as fields inside the same key. typingStore is NOT persisted — it resets on page load.

settingsStore skeleton

import { create } from "zustand";
import { devtools, persist } from "zustand/middleware";
import type { ISettingsState, ISettingsStore } from "./ISettingsStore";

const initial: ISettingsState = {
  visualMode: "time", duration: 30, lang: "en",
  caretStyle: "line", suddenDeath: false, muted: false,
  history: [], best: null,
};

const useSettingsStore = create<ISettingsStore>()(
  devtools(persist(set => ({
    ...initial,
    setVisualMode: (vm) => set({ visualMode: vm }),
    setDuration:   (d)  => set({ duration: d }),
    setLang:       (l)  => set({ lang: l }),
    setCaretStyle: (s)  => set({ caretStyle: s }),
    toggleSuddenDeath: () => set(s => ({ suddenDeath: !s.suddenDeath })),
    toggleMute:    () => set(s => ({ muted: !s.muted })),
    saveResult: (result, lang) => set(s => {
      // ... same logic as useStorage.saveResult
    }),
  }), { name: "cybertype_settings" }), { name: "settings-store" })
);
export default useSettingsStore;
